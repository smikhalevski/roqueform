Interface ConstraintValidationMixin

The mixin added to fields by the constraintValidationPlugin.

Properties

Readonly error

error: null | string

An error associated with the field, or null if there's no error.

Readonly isInvalid

isInvalid: boolean

true if the field or any of its derived fields have an associated error, or false otherwise.

Readonly validity

validity: null | ValidityState

The validity state, or null if there's no associated element, or it doesn't support Constraint Validation API.

Methods

clearErrors

  • clearErrors(): void

  • Recursively deletes errors associated with this field and all of its derived fields. Calls setCustomValidity if the field has an associated element.

    If a field has an associated element that doesn't satisfy validation constraints this method is no-op.

    Returns void

deleteError

  • deleteError(): void

  • Deletes an error associated with this field. Calls setCustomValidity if the field has an associated element.

    If a field has an associated element that doesn't satisfy validation constraints this method is no-op.

    Returns void

refCallback

  • refCallback(element): void

  • The callback that associates the field with the DOM element.

    Parameters

    • element: null | Element

    Returns void

reportValidity

  • reportValidity(): boolean

  • Shows error message balloon for the first element that is associated with this field or any of its derived fields, that has an associated error via calling reportValidity.

    Returns boolean

    true if a field doesn't have an error, or false otherwise.

setError

  • setError(error): void

  • Associates an error with the field. Calls setCustomValidity if the field has an associated element.

    If a field has an associated element that doesn't satisfy validation constraints this method is no-op.

    Parameters

    • error: string

      The error to set.

    Returns void
